The current specification is known as the HTML Living Standard and is maintained by the W3C (a consortium of the major browser vendors such as Apple, Google, Mozilla, and Microsoft) and the Web Hypertext Application Technology Working Group (WHATWG).

Semantics

HTML tags provide meaning to the elements it wraps

  • Search engines will consider its contents as important keywords to influence the page's search rankings
  • Screen readers can use it as a signpost to help visually impaired users navigate a page
  • Finding blocks of meaningful code is significantly easier than searching though endless divs with or without semantic or namespaced classes
  • Suggests to the developer the type of data that will be populated
  • Semantic naming mirrors proper custom element/component naming

Content Model

Each HTML element has a content model, its the type of content that the element expects.

Nothing:
The element admits no content, therefore no text and no nodes, for convenience these elements are self closing tags.
Metadata:
Content that defines how the document is going to behave, present itself or relate to other documents or applications.
Flow:
Most of the elements used between the body tags.
Sectioning:
Sectioning content is content that defines the scope of headings and footers.
Headings:
Defines the header of a section.
Phrasing:
Text, as well as the elements that might be used to markup such text inside a parragraph.
Embedded:
Content that imports another resource into the document.
Interactive:
Content that is ment to interact with the user.
Palpable:
Any element that allows for flow or phrasing content should have at least one node and does not have the attribute hidden.
Script supporting:
Do not represent anything by themselves and are used to contain scripts.
Transparent:
The content model of a transparent element is derived from the content model of its parent element.
